# Testing Session Log

## Proposito

Documenta el desarrollo operativo de cada sesion de prueba para conservar contexto, incidentes y decisiones tomadas durante la evaluacion.

## Campos utiles

- Fecha, version y escenario evaluado.
- Participantes y rol de cada uno.
- Tareas ejecutadas y tiempo empleado.
- Incidencias, bloqueos y correcciones aplicadas.
- Observaciones relevantes para analisis posterior.

## Criterios

1. Mantener orden cronologico para reconstruir la sesion.
2. Diferenciar hechos observados de interpretaciones.
3. Guardar trazabilidad suficiente para repetir la prueba.
4. Referenciar cualquier cuestionario o checklist usado en la sesion.
