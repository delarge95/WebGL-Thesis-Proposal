# External Docs

Repositorio de referencias externas utilizadas como apoyo académico, visual o técnico.

## Regla de uso

- Esta carpeta no debe recibir documentación fuente del proyecto.
- Todo documento creado por el proyecto debe vivir en su familia documental correspondiente.
- Las referencias externas se organizan por tipo para evitar mezclar plantillas, bibliografía operativa y guías técnicas.

## Subcarpetas

- `academic_templates/`: formatos, formularios, plantillas y normas académicas externas.
- `ux_ui_reference/`: guías y recursos visuales externos de UX/UI.
- `unity_webgl_reference/`: guías externas de Unity, UI Toolkit y WebGL.

## Regla adicional de clasificación

- Si un documento analiza iteraciones, problemas, rediseños o decisiones de este proyecto, no pertenece aquí aunque trate sobre UX/UI.
- Ese tipo de material debe vivir en `Informe_final/Desarrollo_App/` o en la capa documental interna que corresponda.

## Relación con otras carpetas documentales

- `desarrollo/docs/`: documentación técnica e investigativa producida durante el desarrollo.
- `docs/manuals/`: manuales y documentos listos para publicación o consulta operativa.
- `Informe_final/`: entregables formales de tesis y defensa.
- `Informe_final/Desarrollo_App/`: bitácora, auditoría, consolidaciones y control de reestructuración.
