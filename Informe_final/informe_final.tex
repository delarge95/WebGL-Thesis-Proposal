\section*{Resumen}
\addcontentsline{toc}{section}{Resumen}
\setlength{\parindent}{0pt}
La documentación de hardware técnico complejo, como drones y otros sistemas electromecánicos multicomponente, enfrenta un desafío persistente de comunicación espacial cuando depende de imágenes 2D y documentos PDF estáticos. Este informe desarrolla y documenta la construcción de un prototipo web 3D interactivo basado en Unity Web para la visualización técnica, inspección y análisis visual del ensamblaje del dron Holybro X500~V2. El marco teórico integra la Teoría de la Carga Cognitiva como lente interpretativa, principios de interacción humano-computador en entornos 3D y fundamentos de renderizado físicamente basado. Metodológicamente, el trabajo se estructura como investigación aplicada con enfoque mixto y predominio cualitativo-formativo, enmarcado en Design Science Research, prototipado iterativo y evaluación comparativa frente a un soporte 2D de referencia. La build funcional integra selección de piezas, \textit{bottom sheet} de detalle, modos \texttt{Inspect}/\texttt{Analyze}/\texttt{Studio}, corte transversal, \textit{explode}, filtros por categoría y una visualización térmica híbrida por componentes. La arquitectura final articula UI Toolkit, gestores creados en \textit{runtime}, saneamiento del modelo importado mediante \texttt{ImportedDroneRuntimeBinder} y un subsistema térmico heurístico coherente con las restricciones de Unity Web y WebAssembly. La validación contempla \textit{profiling} técnico, desempeño en tareas, SUS aplicado al prototipo 3D, NASA-TLX Raw por condición y protocolo \textit{Think-Aloud}; sin embargo, el cierre empírico definitivo del capítulo de resultados depende de la build congelada posterior a la optimización final del \textit{pipeline} CAD a Unity.

\hspace{1.27cm}\textit{\textbf{Palabras clave:}} WebGL, Unity, WebAssembly, visualización técnica 3D, UI Toolkit, optimización CAD a WebGL, carga de trabajo percibida, NASA-TLX, \textit{Technical Art}, hardware complejo.

\section*{Abstract}
\addcontentsline{toc}{section}{Abstract}
\setlength{\parindent}{0pt}
Complex hardware documentation, especially for multicomponent electromechanical systems such as drones, still faces persistent spatial communication challenges when it relies on static 2D images and PDF documents. This report develops and documents an interactive 3D web prototype based on Unity Web for the technical visualization, inspection, and visual analysis of the Holybro X500~V2 assembly. The theoretical framework combines cognitive load theory as an interpretive lens, human-computer interaction in 3D environments, and physically based rendering principles. Methodologically, the work is framed as applied research with a mixed-method approach and qualitative-formative predominance, supported by Design Science Research, iterative prototyping, and comparative evaluation against a 2D reference medium. The functional build integrates part selection, a bottom detail sheet, \texttt{Inspect}/\texttt{Analyze}/\texttt{Studio} modes, cross-section, exploded view, category filters, and a hybrid component-based thermal visualization. The final architecture combines UI Toolkit, runtime-created managers, import repair through \texttt{ImportedDroneRuntimeBinder}, and a heuristic thermal subsystem aligned with the constraints of Unity Web and WebAssembly. Validation is structured around technical profiling, task performance, SUS applied to the 3D prototype, condition-specific NASA-TLX Raw, and a Think-Aloud protocol; however, the definitive empirical closure of the results chapter still depends on the frozen build obtained after the final CAD-to-Unity optimization pass.

\hspace{1.27cm}\textit{\textbf{Keywords:}} WebGL, Unity, WebAssembly, technical 3D visualization, UI Toolkit, CAD-to-WebGL optimization, perceived workload, NASA-TLX, Technical Art, complex hardware.
