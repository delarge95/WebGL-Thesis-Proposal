# desarrollo/docs

This folder is the working technical documentation layer of the repository.

## Purpose

- Capture architecture, setup, optimization, pipeline, and research material produced during development.
- Preserve the technical reasoning behind implementation decisions.
- Serve as the source workspace for documents that may later be mirrored into the published documentation layer.

## What belongs here

- Internal technical guides such as `UNITY_SETUP.md`, `WEBGL_BUILD_SETTINGS.md`, and optimization notes.
- Architecture and deployment documentation used during active implementation.
- Research and comparative analyses under folders such as `investigacion/` and `diseno/`.

## What does not belong here

- Third-party references or templates.
- GitHub Pages assets or site bundles.
- Formal thesis packaging, defense material, or audit manifests.

## Relationship with other documentation folders

- `desarrollo/docs/` is for working technical authorship.
- `docs/` is for published or publishable documentation.
- `External_docs/` is for material authored outside the project.
- `Informe_final/` is for the academic delivery package.

When a document is duplicated in `docs/manuals/`, that duplication should be read as a publication mirror, not as a second independent documentation source.
