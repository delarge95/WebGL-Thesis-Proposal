# SUS Benchmark Data

## Proposito

Concentra valores de referencia del System Usability Scale para comparar tendencias entre pruebas, versiones o perfiles de usuario.

## Uso esperado

- Establecer una base de comparacion para interpretacion rapida.
- Señalar si la version actual cae por debajo del umbral esperado.
- Apoyar la lectura historica junto a `Usability_Metrics_Report.md`.
- Servir como apoyo para discusiones de remediacion UX.

## Criterios

1. Los benchmarks deben indicar contexto de muestra y fecha.
2. La comparacion debe ser reproducible con la misma escala SUS.
3. No mezclar valores de distintos perfiles sin etiquetarlos.
4. Documentar cualquier ajuste metodologico en la misma nota.
