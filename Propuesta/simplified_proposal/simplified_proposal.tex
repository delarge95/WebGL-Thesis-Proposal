% --- RESUMEN ---
\section*{Resumen}
\phantomsection
\addcontentsline{toc}{section}{Resumen}
\setlength{\parindent}{0pt}
La documentaci\'on t\'ecnica de hardware complejo, como drones multicomponente, enfrenta un l\'imite claro cuando depende de im\'agenes 2D y PDF est\'aticos: el usuario debe reconstruir mentalmente las relaciones espaciales entre piezas, lo que aumenta la carga cognitiva y reduce la capacidad de inspecci\'on. Este proyecto propone una soluci\'on concreta: un visor web 3D interactivo, construido con Unity y desplegado en el navegador mediante WebGL y WebAssembly, aplicado al dron Holybro X500~V2.

El marco te\'orico se apoya en tres pilares: teor\'ia de la carga cognitiva (Sweller, 1988), principios de interacci\'on humano-computador en entornos 3D y renderizado f\'isicamente basado. El enfoque metodol\'ogico es \textit{Design Science Research} (DSR), que orienta la construcci\'on iterativa del artefacto y su evaluaci\'on formativa con usuarios reales.

La evaluaci\'on combina medici\'on de rendimiento t\'ecnico (FPS, carga, uso de memoria), desempe\~no en tareas, usabilidad percibida mediante el cuestionario SUS y carga de trabajo percibida con NASA-TLX, contrastando el visor 3D con un soporte 2D de referencia. El resultado esperado es documentar bajo qu\'e condiciones t\'ecnicas y metodol\'ogicas una aplicaci\'on web 3D resulta operativamente viable para la visualizaci\'on t\'ecnica en contexto acad\'emico e ingenieril.

\hspace{1.27cm}\textit{\textbf{Palabras clave:}} WebGL, Unity, WebAssembly, Renderizado F\'isicamente Basado (PBR), Visualizaci\'on 3D Interactiva, Optimizaci\'on de Assets 3D, Carga de Trabajo Percibida, Interacci\'on Humano-Computador.

% --- ABSTRACT ---
\section*{Abstract}
\phantomsection
\addcontentsline{toc}{section}{Abstract}
\setlength{\parindent}{0pt}
Technical documentation of complex hardware, such as multi-component drones, faces a clear limitation when it relies on static 2D images and PDF files: users must mentally reconstruct spatial relationships between parts, which increases cognitive load and reduces inspection accuracy. This project proposes a concrete solution: an interactive 3D web viewer, built with Unity and deployed in the browser via WebGL and WebAssembly, applied to the Holybro X500~V2 drone.

The theoretical framework rests on three pillars: cognitive load theory (Sweller, 1988), human-computer interaction principles in 3D environments, and physically based rendering. The methodological approach is Design Science Research (DSR), which guides the iterative construction of the artifact and its formative evaluation with real users.

Evaluation combines technical performance measurement (FPS, load time, memory usage), task performance, perceived usability through the SUS questionnaire, and perceived workload through NASA-TLX, contrasting the 3D viewer against a 2D reference medium. The expected outcome is to document the technical and methodological conditions under which a 3D web application is operationally viable for technical visualization in academic and engineering contexts.

\hspace{1.27cm}\textit{\textbf{Keywords:}} WebGL, Unity, WebAssembly, Physically-Based Rendering (PBR), Interactive 3D Visualization, 3D Asset Optimization, Perceived Workload, Human-Computer Interaction.
