# User Journey Mapping

## Proposito

Visualiza el recorrido del usuario para identificar pasos criticos, fricciones y momentos donde la experiencia necesita apoyo.

## Elementos a mapear

- Objetivo del usuario.
- Paso o accion principal.
- Dificultad o riesgo.
- Resultado esperado.

## Criterios

1. El mapa debe reflejar el flujo real, no el idealizado.
2. Debe mostrar donde la interfaz ayuda o entorpece.
3. La nota debe servir para priorizar mejoras.
4. Tiene que enlazar con hallazgos cualitativos y pruebas de sesion.
