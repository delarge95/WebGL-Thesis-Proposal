\section{Resultados y Análisis}

Este capítulo permanece deliberadamente abierto hasta completar el congelamiento del build final y la ejecución de las mediciones definidas en la metodología. Con el fin de evitar resultados aparentes o plantillas metodológicas, aquí solo se fija el criterio de cierre del capítulo.

\subsection{Condiciones para cerrar el capítulo}

El capítulo 5 solo debe consolidarse cuando existan, en la misma versión congelada del producto, los siguientes insumos:

\begin{enumerate}
    \item build WebGL final posterior a la optimización CAD/import;
    \item rerun de auditorías de jerarquía y cobertura sobre la escena congelada;
    \item KPIs técnicos medidos sobre esa build;
    \item aplicación de SUS, NASA-TLX Raw y protocolo \textit{Think-Aloud};
    \item tablas y figuras generadas a partir de datos reales, no de plantillas.
\end{enumerate}

\subsection{Resultados técnicos pendientes de medición}

Deben reportarse con valores reales, obtenidos del build congelado:

\begin{itemize}
    \item frame rate y frame time;
    \item tiempos de carga;
    \item draw calls;
    \item memoria;
    \item presupuesto geométrico y consistencia funcional final.
\end{itemize}

\subsection{Resultados de usabilidad pendientes de ejecución}

La evaluación con usuarios debe incluir:

\begin{itemize}
    \item puntaje SUS agregado y su interpretación;
    \item puntaje NASA-TLX Raw y lectura por dimensiones;
    \item hallazgos cualitativos del protocolo \textit{Think-Aloud};
    \item discusión integradora frente a los objetivos del proyecto.
\end{itemize}

\subsection{Nota editorial}

En esta versión saneada del informe se eliminaron las plantillas y tokens de relleno del capítulo 5 para evitar confundir estructura metodológica con resultados observados. La versión final de este capítulo debe construirse únicamente con datos reproducibles y fechados sobre la build definitiva.
