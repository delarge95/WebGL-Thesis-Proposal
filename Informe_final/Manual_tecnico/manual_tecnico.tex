\section{Introduccin}
Este documento tcnico detalla la arquitectura, componentes y procedimientos de mantenimiento del prototipo Web 3D para visualizacin tcnica de drones. El sistema implementa 104 scripts C\# (~16,927 lneas), 9 shaders personalizados (~1,749 lneas), 5 hojas de estilo USS (~3,561 lneas) y 4 layouts UXML (~502 lneas), con arquitectura modular, patrones de diseo profesionales y optimizaciones especficas para WebGL.

\subsection{Alcance del Documento}
\begin{itemize}
    \item Arquitectura del sistema y patrones de diseo
    \item Descripcin de todos los sistemas implementados
    \item Procedimientos de compilacin y despliegue
    \item Guas de extensin y mantenimiento
\end{itemize}

\section{Arquitectura del Sistema}

\subsection{Stack Tecnolgico}
\begin{itemize}
    \item \textbf{Motor:} Unity 6000.0.62f1 (Unity 6)
    \item \textbf{Lenguaje:} C\# 11 (Scripting) / HLSL (Shaders)
    \item \textbf{Plataforma:} WebGL 2.0
    \item \textbf{Compilador:} IL2CPP $\rightarrow$ WebAssembly (WASM)
    \item \textbf{Pipeline Grfico:} Universal Render Pipeline (URP)
    \item \textbf{UI Framework:} UI Toolkit (USS/UXML)
\end{itemize}

\subsection{Patrones de Diseo Implementados}
\begin{itemize}
    \item \textbf{Singleton/PersistentSingleton:} Para managers globales
    \item \textbf{Service Locator (H01):} Resolucin de dependencias en tiempo de ejecucin
    \item \textbf{Event Bus:} Comunicacin desacoplada con deteccin de leaks y purga de zombies (H03)
    \item \textbf{Strategy (BaseModeHandler, H02):} 3 handlers concretos (InspectHandler, AnalyzeHandler, StudioHandler)
    \item \textbf{State Machine:} Control de estados de la aplicacin (9 estados)
    \item \textbf{ScriptableObjects:} Configuracin de datos (DronePartData, AssemblyGuideData)
\end{itemize}

\subsubsection{Subsistema Trmico Hbrido (V1)}
Se incorpor una base de simulacin trmica hbrida orientada a WebGL mvil. Esta arquitectura combina un solver reducido por componentes, una capa de visualizacin espacial por pieza y un pipeline de verificacin matemtica apoyado por WolframAlpha. Los mdulos principales son:
\begin{itemize}
    \item \texttt{ThermalSimulationManager}: solver por componentes con temperatura ambiente, carga sostenida y conduccin heurstica.
    \item \texttt{ThermalViewController}: puente entre simulacin y render mediante \texttt{MaterialPropertyBlock}.
    \item \texttt{ThermalSurfaceProfile}: configuracin opcional por pieza crtica para hotspot, direccin y propagacin visual.
    \item \texttt{ThermalContactGraphAsset}: base serializable del grafo de contactos trmicos.
    \item \texttt{ThermalContactGraphBuilderWindow}: herramienta de editor para generar el grafo trmico offline.
\end{itemize}



Esta V1 no debe describirse todavia como FEA en tiempo real ni como termografia validada experimentalmente; su proposito es ofrecer consistencia fisica y credibilidad visual dentro de las restricciones de Unity WebGL.

En el refinamiento final de esta etapa, la capa visual termica paso a usar un shimmer mas suave, de baja frecuencia y amplitud reducida, mientras que la prioridad perceptual se mantiene en motores, ESC, bateria y electronica central. El incremento de piezas y subpiezas visibles del modelo no redefine la jerarquia termica oficial; solo aumenta la granularidad visual del ensamblaje.

La ruta oficial de esta V1 ya usa un grafo de contactos explicito (\texttt{ThermalCanonicalContactGraph.asset}) y una leyenda termica visible en modo \texttt{Thermal}. En esta rama, la carga termica sigue gobernada por \texttt{DroneStateController.SystemLoadFactor} y por los estados del dron; un slider de usuario dedicado permanece como mejora futura. El archivo \texttt{ThermalTestSetup.cs} se conserva unicamente como harness experimental para CAD bruto y no forma parte del pipeline oficial.

Configuracin completa de cada pieza del Holybro X500~V2 (16 piezas con 32+ campos cada una):
\begin{itemize}

    \item Vista Explosionada: direccin (\texttt{Vector3}), distancia (0.3--2.5), prioridad (1--10)
    \item Info de Ensamblaje: herramientas, torque (Nm), tornillos, dificultad (1--5), tiempo (min), prerequisitos
    \item Perfil Trmico: color de highlight para shader Thermal (azul\,=\,fro a rojo\,=\,caliente)
\end{itemize}

\section{Compilacin y Despliegue}

\subsection{Requisitos de Build}
\begin{enumerate}
    \item Unity 6.3 LTS
    \item WebGL Build Support instalado
    \item URP configurado
\end{enumerate}

\subsection{Configuracin de Build}
\begin{enumerate}
    \item File $>$ Build Settings $>$ WebGL
    \item Player Settings:
    \begin{itemize}
        \item Compression: Brotli (produccin) o Disabled (desarrollo)
        \item Code Stripping: High
        \item IL2CPP Code Generation: Faster (Smaller) Builds
    \end{itemize}
\end{enumerate}

\subsection{Despliegue}
Subir carpeta Build/ a servidor con headers:
\begin{verbatim}
Content-Type: application/wasm (para .wasm)
Content-Encoding: br (si usa Brotli)
\end{verbatim}

\section{Extensin del Sistema}

\subsection{Aadir Nueva Pieza}
\begin{enumerate}
    \item Crear DronePartData asset (Create $>$ WebGL $>$ Drone Part Data)
    \item Configurar todos los campos
    \item Aadir ExplodablePart component al modelo 3D
    \item Asignar DronePartData al component
    \item Aadir ConnectionPointMarker a puntos de conexin
\end{enumerate}

\subsection{Aadir Nuevo Modo de Vista}
\begin{enumerate}
    \item Aadir entrada a enum ViewMode
    \item Crear material en ViewModeManager
    \item Implementar lgica en ApplyModeToRenderers()
    \item Aadir botn en ViewModeToolbar
\end{enumerate}

\subsection{Aadir Nuevo Manager}
\begin{enumerate}
    \item Heredar de Singleton<T> o PersistentSingleton<T>
    \item Implementar lgica en Awake() y Start()
    \item Suscribirse a eventos relevantes del EventBus
    \item Aadir al GameObject \_GameManagers en escena
\end{enumerate}
