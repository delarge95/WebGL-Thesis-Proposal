# Índice y Organización del Portafolio

## Objetivo

Usar esta carpeta como espacio personal de organización para preparar piezas de portafolio derivadas del proyecto sin alterar la fuente de verdad técnica o académica.

## Estructura actual

- `documentacion/`: planeación, índices, checklists, guiones y breakdowns.
- `assets_fuente/`: capturas, renders, GIFs, clips, comparativas y exportaciones finales.
- `herramientas_fuente/`: scripts, shaders, tooling y extractos técnicos reutilizables.

## Regla principal

- Los archivos fuente del proyecto siguen viviendo en sus rutas originales.
- Esta carpeta no sustituye la documentación técnica ni la tesis; sólo organiza el material que luego se convertirá en piezas de portafolio.

## Secuencia de trabajo sugerida

1. Definir pieza objetivo.
2. Identificar fuentes en el proyecto original.
3. Reunir en esta carpeta sólo el material necesario para presentación.
4. Redactar breakdown técnico breve.
5. Producir versión final para ArtStation, LinkedIn, reel o README curado.

## Documentos base

- `08_Portafolio_Tech_Artist.md`: documento maestro estratégico.
- `01_Checklist_Ejecutivo_Portafolio.md`: ejecución táctica inmediata.
- `02_Fuentes_y_Piezas_Candidatas.md`: mapa de qué extraer y desde dónde.

## Reglas de clasificación interna

- `documentacion`: textos y organización.
- `assets_fuente`: material visual listo para curar.
- `herramientas_fuente`: material técnico demostrable.

## Restricciones actuales

- No intervenir `blender_files/` mientras siga en preparación de meshes para Unity.
- Mantener Holybro como lote técnico activo; extraer para portafolio sólo por copia o staging, no por vaciado del origen.
