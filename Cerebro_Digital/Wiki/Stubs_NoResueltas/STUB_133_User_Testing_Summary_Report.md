# User Testing Summary Report

## Proposito

Sintetiza los hallazgos de una bateria de pruebas con usuarios para comunicar conclusiones, fricciones y prioridades de ajuste.

## Contenido minimo

- Objetivo de la sesion y perfil de participantes.
- Tareas clave evaluadas.
- Problemas mas repetidos y su impacto.
- Hallazgos positivos que conviene preservar.
- Recomendaciones priorizadas para la remediacion.

## Criterios

1. El resumen debe poder leerse sin revisar la bitacora completa.
2. Las conclusiones deben enlazar a evidencia concreta.
3. La priorizacion de problemas debe ser expliicta y defendible.
4. Debe quedar claro que partes requieren seguimiento tecnico o de UX.
