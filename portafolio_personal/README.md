# Portafolio Personal

Carpeta de curaduría personal para extraer piezas de portafolio a partir de este proyecto sin mezclar ese material con la tesis, el build WebGL o la documentación técnica operativa.

## Reglas de uso

- Herramientas generadas para el pipeline Holybro:
  - `desarrollo/docs/investigacion/Holybro/cad_symmetry_addon_v5_ultimate.py`
  - `desarrollo/docs/investigacion/Holybro/cad_symmetry_addon_v6_batch.py`
  - `desarrollo/docs/investigacion/Holybro/generate_inventory.py`

## Subcarpetas

- `documentacion/`: documentos guía, organización, checklists, breakdowns y notas de presentación.
- `assets_fuente/`: staging para recopilar capturas, renders, GIFs, videos o exportaciones finales.
- `herramientas_fuente/`: staging para reunir scripts, shaders, tooling y extractos técnicos reutilizables en piezas de portafolio.

## Estructura inicial de `assets_fuente/`

- `assets_fuente/heroes/`: imágenes de portada, banners y thumbnails principales.
- `assets_fuente/screenshots/`: capturas estáticas del visor, shaders y comparativas.
- `assets_fuente/clips/`: video corto, GIFs o recortes para reel y publicaciones.
- `assets_fuente/diagramas/`: diagramas del pipeline, arquitectura o flujos visuales.
- `assets_fuente/README.md`: criterio de uso y destino esperado de cada subcarpeta.

## Documentos organizativos iniciales

- `documentacion/00_Indice_y_Organizacion.md`: mapa de la carpeta y reglas operativas.
- `documentacion/01_Checklist_Ejecutivo_Portafolio.md`: checklist accionable de publicación, reel y perfil profesional.
- `documentacion/02_Fuentes_y_Piezas_Candidatas.md`: inventario curado de activos, herramientas y piezas que conviene extraer del proyecto.
- `documentacion/03_Checklist_Maestro_Reestructuracion.md`: secuencia de fases y subtareas para continuar la reestructuración sin perder trazabilidad.
- `documentacion/04_Breakdown_Pipeline_CAD_a_WebGL.md`: resumen corto del flujo técnico desde CAD industrial hasta runtime WebGL, orientado a ArtStation, reel y entrevistas.
- `documentacion/05_Breakdown_Sistema_Visualizacion_Interactiva.md`: resumen corto del sistema interactivo del visor, sus overlays técnicos y su valor como pieza de producto.
- `documentacion/06_Plan_Activos_Visuales_Portafolio.md`: shot list y lista de entregables visuales para transformar los breakdowns en material publicable.
- `documentacion/07_Breakdown_Sistema_Termico_Hibrido.md`: breakdown tecnico vivo del sistema termico hibrido, ya alineado con el grafo canonico, la leyenda UI y la frontera entre flujo oficial y tooling experimental.
- `assets_fuente/README.md`: guía mínima para almacenar y clasificar capturas, clips, heroes y diagramas.
- `herramientas_fuente/README.md`: manifiesto de la curaduría técnica inicial y listado de scripts, shaders y componentes ya staged.
