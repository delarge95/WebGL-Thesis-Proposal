\section{Introduccion}
Este manual describe el uso de la build final documentada del visor WebGL del Holybro X500~V2. El documento se limita al flujo visible de la aplicacion y evita describir herramientas ocultas, experimentales o no integradas en la interfaz final.

\subsection{Alcance}
La aplicacion permite:
\begin{itemize}
    \item ingresar desde el Hero al visor principal;
    \item navegar el modelo 3D;
    \item seleccionar piezas y consultar su ficha tecnica en un \textit{bottom sheet};
    \item usar herramientas de inspeccion, analisis y visualizacion.
\end{itemize}

\subsection{Publico objetivo}
\begin{itemize}
    \item estudiantes e investigadores de ingenieria;
    \item tecnicos o docentes que requieran lectura visual del ensamblaje;
    \item evaluadores del proyecto de grado.
\end{itemize}

\section{Requisitos y acceso}

\subsection{Requisitos minimos}
\begin{itemize}
    \item navegador moderno con soporte WebGL~2.0;
    \item conexion estable para cargar la aplicacion;
    \item mouse o pantalla tactil.
\end{itemize}

\subsection{Ingreso}
\begin{enumerate}
    \item abrir la URL publica del prototipo o el build local de prueba;
    \item esperar la carga inicial;
    \item usar el Hero para revisar el proyecto o ingresar con el boton \texttt{EXPLORE}.
\end{enumerate}

\section{Flujo general de uso}

El recorrido visible del sistema es:

\begin{center}
\texttt{Hero -> Explore -> seleccion de pieza -> bottom sheet -> Inspect / Analyze / Studio}
\end{center}

\begin{itemize}
    \item El boton \texttt{HOME} regresa al Hero.
    \item El boton \texttt{RESET VIEW} restablece la vista principal.
    \item La tecla \texttt{Esc} cierra primero la ficha inferior o el subpanel activo, si existe uno abierto.
\end{itemize}

\section{Seleccion de piezas y ficha tecnica}

\begin{enumerate}
    \item hacer clic sobre una pieza visible del dron;
    \item el sistema resalta la seleccion;
    \item la informacion aparece en una ficha inferior (\textit{bottom sheet});
    \item la ficha muestra identificacion, funcion, material, peso, dimensiones y datos de ensamblaje cuando existen.
\end{enumerate}

\noindent La ficha inferior es el panel oficial de consulta. Este manual no describe un panel lateral derecho porque ese flujo no corresponde a la version final documentada.

\section{Herramientas visibles}

\subsection{Modo Inspect}
El modo \texttt{Inspect} reune herramientas de lectura puntual:
\begin{itemize}
    \item \textbf{Pins:} activa o desactiva hotspots;
    \item \textbf{Isolate:} a\'isla la seleccion actual;
    \item \textbf{Power:} enciende o apaga el estado operativo del dron;
    \item \textbf{Load slider:} ajusta el factor de carga asociado al estado de potencia.
\end{itemize}

\subsection{Modo Analyze}
El modo \texttt{Analyze} concentra herramientas de lectura estructural:
\begin{itemize}
    \item \textbf{Cut:} habilita el corte transversal;
    \item \textbf{Explode:} separa el ensamblaje;
    \item \textbf{Filter:} activa o desactiva categorias funcionales.
\end{itemize}

\noindent Las categorias activas en la build final son:
\begin{itemize}
    \item \texttt{SkeletonAirframe}
    \item \texttt{PropulsionSystem}
    \item \texttt{Avionics}
    \item \texttt{SensorsComms}
    \item \texttt{PowerDistribution}
    \item \texttt{Fasteners}
\end{itemize}

\subsection{Modo Studio}
El modo \texttt{Studio} agrupa visualizacion y ambiente:
\begin{itemize}
    \item \textbf{Render mode:} el estado base es \texttt{Realistic}; la UI visible expone \texttt{X-Ray}, \texttt{Solid Color} y \texttt{Thermal};
    \item \textbf{Environment:} tarjetas \texttt{Studio}, \texttt{Time} y \texttt{Color};
    \item \textbf{Sliders:} rotacion e intensidad de luz.
\end{itemize}

\noindent Cuando se activa \texttt{Thermal}, aparece la leyenda termica visible en pantalla.

\section{Corte transversal}

\begin{enumerate}
    \item abrir \texttt{Analyze};
    \item pulsar \texttt{Cut};
    \item elegir el eje y mover el deslizador del plano;
    \item volver al panel principal cuando termine la inspeccion.
\end{enumerate}

\noindent El corte puede permanecer activo hasta que el usuario lo desactive o use \texttt{RESET VIEW}.

\section{Vista explosionada}

\begin{enumerate}
    \item abrir \texttt{Analyze};
    \item pulsar \texttt{Explode};
    \item ajustar el nivel de separacion con el deslizador;
    \item volver a pulsar o llevar el valor a cero para regresar al ensamblaje.
\end{enumerate}

\section{Limitaciones visibles del cierre}

\begin{itemize}
    \item la herramienta de medicion existe en codigo, pero no esta expuesta en la UI final;
    \item el catalogo de piezas y el panel de configuracion no forman parte del flujo visible documentado;
    \item los efectos de audio no hacen parte de la version funcional cerrada al momento de este manual;
    \item las pruebas de usabilidad y reportes finales deben ejecutarse sobre la build congelada posterior a la optimizacion final.
\end{itemize}

\section{Solucion de problemas}

\subsection{La aplicacion no carga}
\begin{itemize}
    \item verificar conexion y compatibilidad WebGL;
    \item recargar la pagina;
    \item probar en un navegador actualizado.
\end{itemize}

\subsection{No puedo cerrar un panel}
\begin{itemize}
    \item usar el boton de cierre de la ficha;
    \item presionar \texttt{Esc};
    \item usar \texttt{RESET VIEW} si hay una herramienta de analisis activa.
\end{itemize}

\subsection{El rendimiento baja}
\begin{itemize}
    \item cerrar otras pesta\~nas pesadas;
    \item esperar a que termine la carga del modelo;
    \item usar los modos visibles del sistema y evitar navegacion excesiva durante la carga inicial.
\end{itemize}
