# Mapa runtime de piezas, hotspots y fasteners

Fuente: `x500v2_parts_data.json`, `holybro_selection_hierarchy.json`, `x500v2_blender_synced_parts.json` y `holybro_fastener_instances.json`.

Notas de control:
- `pdb_runtime_proxy`, `rc_receiver_runtime_proxy` y las ESC `BLHeli-S 20A` eran proxies sintetizados para cubrir entradas del catálogo canónico anterior; no provienen del FBX final y quedan suprimidos en runtime.
- La prioridad por pieza madre sí está serializada en `holybro_fastener_instances.json` mediante `parentCanonicalPartId`.
- La prioridad por subpieza no está serializada todavía; la app la resuelve en runtime por contacto/bounds para la capa `subpieza + fasteners asociados`. Donde no hay una única subpieza canónica, la tabla no inventa asignaciones.
